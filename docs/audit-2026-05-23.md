- **Fail-closed:** Correcto. Si Hermes no está configurado (`HERMES_ENABLED=false`), se salta sin error. Si AgenticPlug no responde, pasa a local. Si local no responde, 503.
- **Secrets:** Correcto. `HERMES_API_KEY` nunca se imprime en logs. Las respuestas al cliente incluyen `mode_used` y `fallback_chain` sin valores sensibles.
- **Timeout:** `UPSTREAM_TIMEOUT_S=60` cubre cada llamada individual. No hay timeout global del request — si los 3 backends tardan 60s cada uno, el request puede durar 180s.
- **Código:** Limpio. Con tipos (`Union`, `List`, `Optional`). Sin dependencias circulares. Estructura plana (2 módulos + Dockerfile + requirements).

## 3. Bugs activos (3 críticos, 1 no crítico)

### Bug #1 — SearxNG crash loop (CRÍTICO)
**Archivo:** `searxng/settings.yml`
**Líneas afectadas:** 14, 15, 16, 17, 22, 23, 24, 42, 46, 47, 58, 59, 60
**Causa:** SearxNG 2026.x rechaza valores `bool` (`false`, `true`) en campos que esperan `str` o `int`. El mensaje exacto es `Expected str, got bool`.
**Valores afectados:** `general.privacypolicy_url`, `general.donation_url`, `general.contact_url`, `general.enable_metrics`, `general.debug`, `brand.public_instances`, `brand.wiki_url`, `brand.issue_url`, `server.base_url`, `server.limiter`, `server.image_proxy`, `ui.query_in_title`, `ui.infinite_scroll`, `ui.center_alignment`.
**Fix:** Cambiar todos los `false` y `true` a `""` (strings vacíos) para campos de tipo string, o a `0`/`1` para campos de tipo int.
**Impacto:** SearxNG no arranca → ecoseek-api y agenticplug no arrancan (dependen de `searxng: service_healthy`). Stack completo bloqueado.

### Bug #2 — EcoAgent crash loop (CRÍTICO)
**Archivo:** `.repos/ecoagent/docker/ecoagent.Dockerfile`
**Error:** `/usr/local/bin/python: No module named ecoagent.tool_server`
**Causa:** `pip install -e .` en el Dockerfile no creó el entry point `ecoagent.tool_server` o el `COPY --from=builder` no copió el módulo.
**Impacto:** ecoagent reinicia infinitamente. No bloquea el stack principal (solo agenticplug depende de ecoagent, y es opcional), pero inutiliza las herramientas ecológicas.

### Bug #3 — ecoseek-api y agenticplug nunca arrancan (CRÍTICO)
**Causa:** Ambos servicios tienen `depends_on: searxng: condition: service_healthy`. Como SearxNG está en crash loop, nunca llegan a `healthy` → ecoseek-api y agenticplug se quedan en `Created`.
**Impacto:** El stack Docker completo no es funcional. No se puede probar el backend gateway ni el failover.

### Bug #4 — `celery` y `colorama` en requirements.txt (NO CRÍTICO)
**Archivo:** `backend/requirements.txt`
**Problema:** `celery` es un distributed task queue (requiere broker, workers, colas persistentes). El gateway es sin estado — solo rutea requests HTTP. `colorama` es para colores en terminal — en un contenedor Docker sin TTY es peso muerto.
**Impacto:** ~2 MB extra en la imagen. No bloquea nada, pero viola el target de imagen <200MB con dependencias innecesarias.
